Documentation

Laravel on Huawei Cloud — serverless, effortless.

vapor-hw adalah CLI tool untuk men-deploy aplikasi Laravel ke Huawei Cloud FunctionGraph. Mirip dengan Laravel Vapor, dioptimalkan untuk ekosistem Huawei Cloud dengan dukungan penuh RDS, DCS Redis, DMS RabbitMQ, CDN, dan APIG.

FunctionGraph APIG RDS MySQL/PostgreSQL DCS Redis DMS RabbitMQ / Kafka OBS Storage CDN CSMS Secrets

Bagaimana cara kerjanya?

vapor-hw mengelola seluruh lifecycle deployment Laravel di Huawei Cloud — mulai dari provisioning infrastruktur hingga deploy dan operasional harian. Kamu hanya perlu satu file konfigurasi (vapor.yaml) dan satu command.

Membuat IAM Agency

FunctionGraph butuh agency agar bisa mengakses service Huawei Cloud lainnya (OBS, RDS, dll). Buat satu kali di console.

1
Buka IAM → Agencies → Create Agency
Agency type: Cloud service, service: FunctionGraph
2
Tambahkan permissions berikut
OBS OperateAccess • RDS FullAccess • DCS FullAccess • DMS FullAccess • CSMS FullAccess • VPC FullAccess • APIG FullAccess
3
Catat nama agency
Nama agency ini diisi di field xrole pada vapor.yaml
⚠️
Pastikan AK/SK yang dipakai memiliki permission yang cukup untuk create semua service (VPC, RDS, DCS, DMS, APIG, OBS, FunctionGraph, CSMS). Gunakan akun dengan policy FullAccess pada semua service tersebut, atau buat custom policy.

Installation

Build vapor-hw dari source dan install ke PATH.

Linux / macOS

bash
# Clone repository
git clone https://github.com/yourorg/vapor-hwc
cd vapor-hwc

# Build binary
go build -o vapor-hw .

# Install ke PATH
sudo mv vapor-hw /usr/local/bin/vapor-hw
sudo chmod +x /usr/local/bin/vapor-hw

Windows (PowerShell)

powershell
cd vapor-hwc
go build -o vapor-hw.exe .

# Pindahkan ke folder yang ada di PATH, atau jalankan langsung:
.\vapor-hw.exe version

Verify instalasi

bash
vapor-hw version
# vapor-hw v0.1.0 (Huawei Cloud)

vapor-hw --help

Quick Start

Deploy Laravel pertama kamu ke Huawei Cloud dalam beberapa langkah.

01
Login dengan Huawei Cloud credentials
Simpan AK/SK secara aman di credential store lokal.
02
Inisialisasi project
Generate vapor.yaml template di root project Laravel.
03
Edit vapor.yaml
Sesuaikan id, region, xrole, dan konfigurasi environment.
04
Set secrets
Upload secret seperti APP_KEY dan DB_PASSWORD ke CSMS.
05
Provision infrastructure
Buat VPC, APIG, RDS, DCS, DMS, OBS bucket secara otomatis.
06
Deploy aplikasi
Build artifact, upload ke OBS, dan deploy ke FunctionGraph.
bash
# 1. Login
vapor-hw login

# 2. Init di root project Laravel
cd /path/to/laravel-project
vapor-hw init

# 3. Edit vapor.yaml sesuai kebutuhan, lalu set secrets
vapor-hw secret set production APP_KEY "base64:xxxxxx"
vapor-hw secret set production DB_PASSWORD "YourPass!"
vapor-hw secret set production REDIS_PASSWORD "RedisPass!"
vapor-hw secret set production RABBITMQ_PASSWORD "RabbitPass!"
vapor-hw secret set production OBS_KEY "your-ak"
vapor-hw secret set production OBS_SECRET "your-sk"

# 4. Provision semua infrastruktur
vapor-hw infra apply

# 5. Deploy!
vapor-hw deploy run production
💡
Semua command bersifat idempoten — aman dijalankan berkali-kali. Jika resource sudah ada, vapor-hw akan memakai yang existing tanpa membuat ulang.

vapor.yaml

File konfigurasi utama yang mendefinisikan seluruh project, infrastruktur, dan environment. Harus berada di root project Laravel.

yaml
id: my-laravel-app      # [WAJIB] prefix semua resource cloud
name: My Laravel App    # [WAJIB] nama tampilan
runtime: PHP8.1         # [WAJIB] PHP7.3|PHP7.4|PHP8.0|PHP8.1|PHP8.2
region: ap-southeast-3  # [WAJIB] region Huawei Cloud
xrole: my-fg-agency     # [WAJIB] nama IAM Agency untuk FunctionGraph

storage:
  bucket: vapor-my-app-storage   # [WAJIB] nama OBS bucket

network:
  vpc:
    name: my-app-vpc
    cidr: 10.0.0.0/16
  subnet:
    name: my-app-subnet
    cidr: 10.0.1.0/24
  security_group:
    name: my-app-sg
    allow_http: true
    allow_https: true

infra:
  apig:
    instance:
      name: my-app-apig
      edition: BASIC     # BASIC|PROFESSIONAL|ENTERPRISE|PLATINUM
      public_access: true
      # instance_id: "abc123"  # isi untuk pakai existing APIG

environments:
  production:
    # ... lihat section Environments

Field utama

FieldWajibKeterangan
id✅Identifier unik project. Huruf kecil, angka, tanda hubung. Dipakai sebagai prefix semua resource Huawei Cloud.
name✅Nama tampilan project (bebas).
runtime✅PHP runtime: PHP7.3 / PHP7.4 / PHP8.0 / PHP8.1 / PHP8.2
region✅Huawei Cloud region. Contoh: ap-southeast-3 (Singapore)
xrole✅Nama IAM Agency untuk FunctionGraph. Buat sekali di IAM Console.
enterprise—Enterprise Project ID. Kosongkan jika tidak pakai.
storage.bucket✅Nama OBS bucket. Harus unik secara global.
ℹ️
Jalankan vapor-hw validate untuk memverifikasi vapor.yaml sebelum deploy. CLI akan mendeteksi field yang hilang atau salah format.

Environments

Setiap environment adalah deployment Laravel yang terpisah dengan resource infra masing-masing. Kamu bisa definisikan sebanyak apapun environment (production, staging, dev, dll).

yaml
environments:
  production:
    domain: api.example.com         # custom domain via APIG
    memory: 1024                    # MB (default: 512, maks: 10240)
    timeout: 300                    # detik (default: 30, maks: 900)
    concurrency: 20                 # max request serentak per instance

    scaling:
      max_instances: 10             # batas auto-scaling
      reserved_instances: 1        # selalu aktif (cegah cold start)

    certificate:
      name: production-ssl
      cert: ./certs/production.crt   # path PEM cert lokal
      key:  ./certs/production.key

    database:   # lihat section Database
    cache:      # lihat section Cache
    queue:      # lihat section Queue
    cdn:        # lihat section CDN

    env:
      APP_ENV: production
      APP_DEBUG: "false"

    secrets:
      - APP_KEY
      - DB_PASSWORD

    build:
      commands:
        - composer install --no-dev --optimize-autoloader
        - php artisan config:cache

    deploy:
      commands:
        - php artisan migrate --force
        - php artisan vapor:health-check

Parameter environment

FieldDefaultKeterangan
domain—Custom domain untuk endpoint Laravel. Jalankan vapor-hw domain [env] setelah deploy pertama untuk bind.
memory512Memory FunctionGraph dalam MB. Rentang: 128–10240, kelipatan 128.
timeout30Request timeout dalam detik. Maks: 900.
concurrency10Max request serentak per instance.
scaling.max_instances5Batas maksimum scaling FunctionGraph.
scaling.reserved_instances0Instance yang selalu aktif. Set ≥ 1 untuk menghindari cold start.
certificate.name—Nama sertifikat SSL di APIG.
certificate.cert—Path ke file PEM certificate di mesin lokal.
certificate.key—Path ke file PEM private key di mesin lokal.

Network (VPC)

VPC, Subnet, dan Security Group dibuat otomatis jika belum ada. Jika sudah ada dengan nama yang sama, akan dipakai (idempoten).

yaml
network:
  vpc:
    name: my-app-vpc
    cidr: 10.0.0.0/16       # bebas, tidak boleh overlap VPC lain
  subnet:
    name: my-app-subnet
    cidr: 10.0.1.0/24      # harus berada dalam CIDR VPC di atas
  security_group:
    name: my-app-sg
    allow_http: true         # buka port 80
    allow_https: true        # buka port 443

API Gateway (APIG)

APIG sebagai entry point HTTP/HTTPS untuk FunctionGraph. Setiap environment mendapat APIG group sendiri dengan domain unik.

Buat APIG baru

yaml
infra:
  apig:
    instance:
      name: my-app-apig
      edition: BASIC        # BASIC | PROFESSIONAL | ENTERPRISE | PLATINUM
      public_access: true

Pakai APIG yang sudah ada

yaml
infra:
  apig:
    instance:
      name: my-existing-apig
      instance_id: "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"  # skip create
ℹ️
Per environment, APIG membuat API group tersendiri dengan /{proxy+} dan / route yang memforward semua request ke FunctionGraph. URL format: https://{env_group_sl_domain}/RELEASE/

Storage (OBS)

OBS bucket digunakan untuk tiga keperluan: artifact deploy, state file infra, dan static asset CDN. Dibuat otomatis jika belum ada.

yaml
storage:
  bucket: vapor-my-app-storage   # nama unik global

Struktur folder di OBS

Path di OBSIsi
{id}/{env}/artifacts/ZIP artifact deploy per version
{id}/{env}/state.jsonState infra (host, port, instance ID, dll)
{id}/assets/{hash}/Static assets dari public/ (CDN)
{id}/{env}/envCustom env vars yang di-push via env push

Integrasi Laravel Filesystem

Untuk menggunakan OBS sebagai filesystem Laravel (Storage::disk('obs')), tambahkan package obs/esdk-obs-php dan konfigurasi di config/filesystems.php:

php (config/filesystems.php)
'obs' => [
    'driver'   => 'obs',
    'key'      => env('OBS_KEY'),
    'secret'   => env('OBS_SECRET'),
    'bucket'   => env('OBS_BUCKET'),
    'endpoint' => env('OBS_ENDPOINT'),
    'region'   => env('OBS_REGION'),
    'prefix'   => env('OBS_PREFIX', ''),  // penting untuk isolasi staging
],
ℹ️
OBS_KEY dan OBS_SECRET adalah AK/SK Huawei Cloud. Simpan sebagai secrets, bukan di env: vapor.yaml.

Database (RDS)

Managed MySQL atau PostgreSQL via Huawei RDS. Database dan user aplikasi dibuat otomatis oleh infra apply.

Opsi A — Buat RDS baru (managed)

yaml
database:
  engine: mysql              # mysql | postgresql
  version: "8.0"             # MySQL: 5.7|8.0 / PostgreSQL: 12|13|14|15
  name: my-app-db-prod       # nama RDS instance
  db_name: laravel           # nama database (dibuat otomatis)
  username: laravel          # user aplikasi (bukan rdsuser)
  password: "ProdDB@Str0ng!"

  flavor: rds.mysql.n1.large.4  # opsional, auto-select jika kosong
  volume_size: 100           # GB (default: 40, min: 40, maks: 4000)
  volume_type: ULTRAHIGH    # COMMON (SATA) | ULTRAHIGH (SSD) | ESSD (NVMe)
  az: ap-southeast-3a
  # port: 3306                # opsional, default MySQL: 3306 / PostgreSQL: 5432

Opsi B — Pakai database external/existing

yaml
database:
  host: 10.10.1.111    # isi host → skip create RDS
  port: 3306
  db_name: laravel
  username: my_app_user
  password: "YourPass!"

Apa yang dilakukan infra apply pada Database?

1
Cek apakah instance sudah ada
Cari berdasarkan name. Jika sudah ada → lanjut ke step 3. Jika belum → create instance baru.
2
Create RDS instance
Tunggu status ACTIVE (5–10 menit). Auto-select flavor jika tidak diset.
3
Ensure database & user
Buat database db_name jika belum ada. Buat user aplikasi jika belum ada. Grant privileges. Idempoten — aman jika sudah ada.
4
Fix MySQL auth plugin (PHP 7.x)
Untuk PHP < 8.1, otomatis set mysql_native_password agar kompatibel dengan ext-pdo_mysql.

Env vars yang di-inject otomatis

env vars injected
DB_CONNECTION=mysql
DB_HOST=10.10.1.xxx       # dari state.json
DB_PORT=3306
DB_DATABASE=laravel
DB_USERNAME=laravel
DB_PASSWORD=xxx           # dari secrets CSMS
⚠️
Jangan simpan DB_PASSWORD di bagian env: vapor.yaml. Gunakan secrets agar nilainya diambil dari CSMS dan tidak tersimpan di OBS state file.

Queue (DMS RabbitMQ)

Managed RabbitMQ via Huawei Distributed Message Service. Digunakan sebagai Laravel queue driver.

Opsi A — Buat DMS RabbitMQ baru

yaml
queue:
  engine: rabbitmq
  version: "3.8.35"
  name: my-app-queue-prod
  username: admin
  password: "ProdRabbit@123!"
  broker_num: 3             # 1=single (dev) | 3/5/7=cluster (prod)
  storage_size: 300         # GB per broker (default: 100)
  storage_type: high        # high (SATA) | ssd (NVMe)
  az: ap-southeast-3a
  # flavor: c6.2u8g.cluster  # opsional, auto-select jika kosong
  port: 5671                 # 5671=AMQPS/SSL (direkomendasikan) | 5672=AMQP
  vhost: /                   # vhost selain "/" dibuat otomatis saat infra apply

Opsi B — Pakai RabbitMQ external/existing

yaml
queue:
  host: 10.10.1.19          # isi host → skip create DMS
  port: 5671
  username: admin
  password: "YourRabbitPass!"
  vhost: /

Env vars yang di-inject otomatis

env vars injected
RABBITMQ_HOST=10.10.1.19
RABBITMQ_PORT=5671
RABBITMQ_USER=admin
RABBITMQ_PASSWORD=xxx     # dari secrets CSMS
RABBITMQ_VHOST=/
QUEUE_CONNECTION=rabbitmq
ℹ️
Port 5671 (AMQPS) adalah port SSL/TLS — direkomendasikan untuk production. Jika kamu set vhost: /staging, vapor-hw akan membuat vhost tersebut secara otomatis via DMS API saat infra apply.

Queue (DMS Kafka)

Managed Apache Kafka via Huawei DMS for Kafka. Pilihan tepat untuk event streaming, high-throughput messaging, dan arsitektur event-driven. Gunakan engine: kafka sebagai alternatif RabbitMQ.

Opsi A — Buat DMS Kafka baru

yaml
queue:
  engine: kafka
  version: "3.x"
  name: my-app-kafka-prod
  username: kafkaadmin
  password: "ProdKafka@123!"
  broker_num: 3                  # 1=single | 3+ = cluster (prod)
  storage_size: 300             # GB total — min 300, dibulatkan ke kelipatan 100
  storage_type: high            # high (SATA) | ssd (NVMe)
  az: ap-southeast-3a
  security_protocol: SASL_SSL  # PLAINTEXT | SASL_SSL | SASL_PLAINTEXT
  sasl_mechanism: PLAIN         # PLAIN | SCRAM-SHA-512
  port: 9093                     # 9092=PLAINTEXT | 9093=SASL_SSL | 9095=SASL_PLAINTEXT
  # vpc_client_plain: false       # true = aktifkan endpoint plaintext di dalam VPC
  # flavor: kafka.2u8g.cluster.small  # opsional, auto-select jika kosong

Opsi B — Pakai Kafka external/existing

yaml
queue:
  engine: kafka
  host: 10.10.1.20              # isi host → skip create DMS
  port: 9093
  username: kafkaadmin
  password: "YourKafkaPass!"
  security_protocol: SASL_SSL
  sasl_mechanism: PLAIN

Security protocol & port

security_protocolPort defaultKapan dipakai
SASL_SSL9093Direkomendasikan untuk production. Auth + enkripsi TLS end-to-end.
SASL_PLAINTEXT9095Auth SASL, koneksi tidak terenkripsi. Hanya untuk koneksi internal VPC yang terpercaya.
PLAINTEXT9092Tanpa auth, tanpa enkripsi. Hanya untuk dev/staging di dalam VPC yang terisolasi.

sasl_mechanism

sasl_mechanismKeterangan
PLAINDefault. Username/password plaintext (aman karena dilindungi TLS di SASL_SSL).
SCRAM-SHA-512Lebih aman, credential di-hash. Diperlukan jika klien tidak mendukung PLAIN.

Env vars yang di-inject otomatis

env vars injected
KAFKA_HOST=10.10.1.20
KAFKA_PORT=9093
KAFKA_BROKERS=10.10.1.20:9093     # host:port siap pakai untuk driver Laravel
KAFKA_USERNAME=kafkaadmin
KAFKA_PASSWORD=xxx                # dari secrets CSMS
KAFKA_SECURITY_PROTOCOL=SASL_SSL
KAFKA_SASL_MECHANISM=PLAIN
⚠️
Berbeda dengan RabbitMQ, vapor-hw tidak otomatis set QUEUE_CONNECTION untuk Kafka. Tambahkan sendiri di bagian env: sesuai package Laravel Kafka yang kamu pakai (misalnya QUEUE_CONNECTION: kafka untuk package mateusjunges/laravel-kafka).

Gunakan package mateusjunges/laravel-kafka atau ensi/laravel-kafka. Konfigurasi di config/kafka.php bisa langsung membaca env vars yang di-inject vapor-hw:

Isolasi multi-environment dengan Kafka

Kafka tidak memiliki konsep vhost seperti RabbitMQ. Isolasi staging dari production dilakukan di level topic prefix dan consumer group di kode aplikasi:

yaml (staging env)
staging:
  queue:
    engine: kafka
    name: my-app-kafka-prod     # instance SAMA dengan production
  env:
    KAFKA_TOPIC_PREFIX: "staging."  # baca di kode: env('KAFKA_TOPIC_PREFIX', '')
    KAFKA_CONSUMER_GROUP: "staging-laravel"

Security group rules yang dibuat otomatis

PortProtocolKeterangan
9092TCPKafka PLAINTEXT
9093TCPKafka SASL+SSL (direkomendasikan)
9095TCPKafka SASL_PLAINTEXT
💡
Semua tiga port Kafka (9092, 9093, 9095) dibuka otomatis oleh infra apply di security group untuk seluruh subnet CIDR. Tidak perlu konfigurasi tambahan.

CDN & Assets

Static asset acceleration via Huawei CDN + OBS. File public/ otomatis diupload setiap deploy dengan hash prefix untuk cache-busting.

yaml
cdn:
  domain: assets.example.com
  cert: ./certs/assets.crt  # opsional, untuk HTTPS CDN domain
  key:  ./certs/assets.key

Setup CDN (sekali saja)

1
Verifikasi kepemilikan domain
Tambah TXT record di DNS provider. vapor-hw menampilkan instruksinya saat infra apply pertama kali.
2
infra apply
CDN domain dibuat, OBS bucket dikonfigurasi dengan bucket policy untuk public read. Cache rules: static files 30 hari, html/php no-cache.
3
Tambah CNAME di DNS
Arahkan domain CDN ke CNAME Huawei yang ditampilkan. Harus DNS only (tidak proxied jika pakai Cloudflare).

Env var CDN yang di-inject

env
ASSET_URL=https://assets.example.com/my-app/assets/ab8fb6a1f743

Laravel asset() dan mix() helper otomatis menggunakan ASSET_URL sebagai base URL.

⚠️
Jika memakai Cloudflare, pastikan CDN domain di-set DNS only (ikon abu-abu). Cloudflare proxy akan konflik dengan SSL certificate Huawei CDN.

Deploying

Deploy aplikasi Laravel ke FunctionGraph dengan satu command.

bash
vapor-hw deploy run production
vapor-hw deploy run staging

# Skip confirmation prompt untuk CI/CD
vapor-hw deploy run production --yes

# Auto-create OBS bucket jika belum ada
vapor-hw deploy run production --auto-create

# List semua versi yang sudah di-deploy
vapor-hw deploy list production

Deploy flow (9 langkah)

1
Build hooks
Jalankan composer install, config:cache, dll di mesin lokal.
2
Package artifact
ZIP seluruh project Laravel (kecuali yang di-ignore), upload ke OBS {id}/{env}/artifacts/.
3
Upload static assets ke CDN
File dari public/ (kecuali .php) diupload ke OBS dengan hash prefix unik. CDN cache dipurge otomatis jika ada perubahan.
4
Update FunctionGraph
FG function diupdate dengan artifact baru, memory/timeout/concurrency, VPC config, dan env vars (DB, Redis, RabbitMQ, ASSET_URL).
5
Inject secrets
Secret dari CSMS di-fetch dan di-inject sebagai environment variables ke FunctionGraph.
6
Buat version baru
FG version dibuat dengan format v{YYYYMMDD}-{HHMMSS}. Alias live diupdate ke version ini.
7
Health check
HTTP GET ke /api/ping endpoint untuk verifikasi FunctionGraph bisa menerima request. Harus return HTTP 200.
8
APIG routing update
Backend APIG diupdate ke version alias live yang baru. Custom domain dikonfirmasi aktif.
9
Deploy hooks
Jalankan php artisan migrate --force, vapor:health-check, dll di FunctionGraph via invoke. Output ditampilkan di terminal.

Flags

FlagKeterangan
--yesSkip semua prompt konfirmasi. Berguna untuk CI/CD pipeline.
--auto-createOtomatis buat OBS bucket jika belum ada.

Build & Deploy Hooks

Jalankan perintah custom sebelum packaging (build) dan setelah deploy selesai (deploy).

yaml
build:
  commands:
    - composer install --no-dev --optimize-autoloader
    - php artisan config:cache
    - php artisan route:cache
    - php artisan view:cache
    - php artisan event:cache

Rollbacks

Kembalikan ke versi deployment sebelumnya dengan cepat. Rollback hanya mengubah alias live FunctionGraph — tidak perlu re-deploy artifact.

bash
# Lihat semua versi yang tersedia
vapor-hw deploy list production
# v20260308-143022  (current)
# v20260307-091501
# v20260305-183022

# Rollback ke versi tertentu
vapor-hw rollback to production v20260307-091501

# Rollback tanpa konfirmasi
vapor-hw rollback to production v20260305-183022 --yes
⚠️
Rollback tidak me-rollback database migrations. Jika migration terbaru membuat perubahan breaking, jalankan vapor-hw db migrate production --rollback --step=1 secara terpisah.

Secrets

Kelola data sensitif via Huawei CSMS (Cloud Secret Manager Service). Secret di-inject otomatis sebagai environment variables saat deploy.

Perintah dasar

bash

vapor-hw secret set production APP_KEY
vapor-hw secret set staging APP_KEY

# Lihat semua secret yang terdaftar
vapor-hw secret list production

# Hapus secret
vapor-hw secret delete production APP_KEY

Daftarkan secret di vapor.yaml

yaml
secrets:
  - APP_KEY          # Laravel application key
  - DB_PASSWORD      # password RDS
  - REDIS_PASSWORD   # password DCS Redis
  - RABBITMQ_PASSWORD # password DMS RabbitMQ (jika engine: rabbitmq)
  - KAFKA_PASSWORD   # password DMS Kafka (jika engine: kafka)
  - OBS_KEY          # AK untuk OBS filesystem
  - OBS_SECRET       # SK untuk OBS filesystem

Penamaan secret di CSMS

Secret disimpan di CSMS dengan format nama: vapor-{id}-{env}-{KEY}

contoh nama di CSMS
vapor-my-laravel-app-production-APP_KEY
vapor-my-laravel-app-production-DB_PASSWORD
vapor-my-laravel-app-staging-APP_KEY
💡
Secret harus di-set sebelum menjalankan vapor-hw deploy run. Secret yang ada di daftar secrets: tapi belum di-set di CSMS akan menyebabkan deploy gagal.

Stream dan filter log dari Huawei LTS (Log Tank Service). Pastikan LTS sudah diaktifkan untuk environment tersebut.

FlagDefaultKeterangan

--limit50Jumlah maksimum baris log yang ditampilkan.
--filter—Filter substring dalam log. Case-insensitive.

Database Operations

Jalankan operasi database via FunctionGraph invoke. Sama seperti Artisan, tapi dengan command yang lebih spesifik untuk database.

bash
# Jalankan migration
vapor-hw db migrate production
vapor-hw db migrate production --force

# Rollback migration
vapor-hw db migrate production --rollback
vapor-hw db migrate staging --rollback --step=2

# Drop semua tabel dan re-run migrations (STAGING ONLY!)
vapor-hw db fresh staging
vapor-hw db fresh staging --seed

# Jalankan database seeders
vapor-hw db seed staging
vapor-hw db seed staging --class=UserSeeder
CommandFlagsKeterangan
db migrate--force, --rollback, --step=NJalankan atau rollback migration. --force untuk bypass konfirmasi production.
db fresh--seedDrop semua tabel + re-run migrations. --seed untuk jalankan seeders setelahnya.
db seed--class=ClassNameJalankan seeders. Tanpa --class, jalankan DatabaseSeeder.
🚨
db fresh akan menghapus semua tabel dan data. Jangan pernah digunakan di environment production. Hanya untuk staging atau development.

Destroy

Hapus environment dan resource terkait. Tidak menghapus infrastruktur shared (VPC, RDS instance, DCS, DMS) — hanya menghapus FunctionGraph function dan APIG binding.

bash
# Preview apa yang akan dihapus (tanpa benar-benar menghapus)
vapor-hw destroy staging --dry-run

# Destroy dengan konfirmasi
vapor-hw destroy staging

# Destroy tanpa konfirmasi (untuk CI/CD)
vapor-hw destroy staging --yes

# Destroy + hapus secrets di CSMS
vapor-hw destroy staging --yes --purge-secrets

# Destroy + hapus artifacts di OBS
vapor-hw destroy staging --yes --purge-secrets --purge-artifacts

Yang dihapus saat destroy

1
Detach APIG ingress route
API group dan backend yang mengarah ke FunctionGraph di-lepas.
2
Delete FunctionGraph function
Semua version dan alias function dihapus.
3
Cleanup CSMS secrets (opsional)
Hapus semua secret vapor-{id}-{env}-* dari CSMS. Hanya jika --purge-secrets.
4
Hapus artifacts OBS (opsional)
Hapus folder {id}/{env}/ di OBS bucket. Hanya jika --purge-artifacts.
5
Archive state file
State file di-rename ke state.archived.json sebagai catatan.
FlagKeterangan
--dry-runPreview perubahan tanpa eksekusi.
--yesSkip prompt konfirmasi.
--purge-secretsHapus juga secrets di CSMS untuk environment ini.
--purge-artifactsHapus juga artifact ZIP di OBS.

Staging + Production

Cara paling efisien: staging berbagi infrastruktur fisik (RDS instance, DCS instance, DMS instance) dengan production, namun tetap terisolasi secara logis.

Pola yang direkomendasikan

yaml (vapor.yaml)
environments:

  production:
    database:
      name: my-app-db           # instance name
      db_name: laravel          # database name
      username: laravel
    cache:
      name: my-app-cache
    queue:
      name: my-app-queue
      vhost: /

  staging:
    database:
      name: my-app-db           # SAMA → pakai instance yang sudah ada
      db_name: laravel_staging  # BEDA → database terpisah
      username: laravel_staging # BEDA → user tidak bisa akses DB prod
    cache:
      name: my-app-cache        # SAMA → pakai instance yang sudah ada
    queue:
      name: my-app-queue        # SAMA → pakai instance yang sudah ada
      vhost: /staging           # BEDA → queue terisolasi dari production
    env:
      REDIS_PREFIX: "staging:"  # prefix semua Redis key staging
      OBS_PREFIX: "staging/"    # subfolder OBS untuk staging files

Isolation Strategy

Detail isolasi per layer untuk memastikan staging tidak mengganggu production.

LayerMekanisme isolasiCara kerja
Databasedb_name berbeda + username berbedainfra apply otomatis create database laravel_staging dan user laravel_staging di instance yang sama. User staging tidak bisa akses database production.
RedisREDIS_PREFIX: "staging:"Bootstrap inject prefix ke semua Redis connection config (default, cache, session). Semua key staging diawali staging: sehingga tidak konflik dengan key production.
RabbitMQvhost: /staginginfra apply otomatis create vhost /staging via DMS API. Queue staging sepenuhnya terpisah dari queue production.
KafkaTopic prefix + consumer groupTidak ada vhost di Kafka. Isolasi via konvensi naming topic (KAFKA_TOPIC_PREFIX) dan consumer group yang berbeda di kode aplikasi.
OBS FilesOBS_PREFIX: "staging/"ObsAdapter membaca prefix dari config, semua file staging disimpan di subfolder staging/ dalam bucket yang sama.
FunctionGraphNama function berbedaProduction: vapor-{id}-production, Staging: vapor-{id}-staging. Function sepenuhnya terpisah.
APIGAPI group berbedaSetiap environment punya APIG group sendiri dengan sl_domain unik.
SecretsNamespace CSMS berbedaProduction: vapor-{id}-production-*, Staging: vapor-{id}-staging-*. Tidak ada sharing.
StateOBS path berbedaProduction: {id}/production/state.json, Staging: {id}/staging/state.json.

Workflow deploy multi-environment pertama kali

bash
# 1. Set secrets untuk production
vapor-hw secret set production APP_KEY "base64:xxx"
vapor-hw secret set production DB_PASSWORD "ProdPass!"
vapor-hw secret set production REDIS_PASSWORD "RedisPass!"
vapor-hw secret set production RABBITMQ_PASSWORD "RabbitPass!"
vapor-hw secret set production OBS_KEY "ak"
vapor-hw secret set production OBS_SECRET "sk"

# 2. Set secrets untuk staging (bisa pakai APP_KEY yang sama)
vapor-hw secret set staging APP_KEY "base64:xxx"
vapor-hw secret set staging DB_PASSWORD "StagingPass!"
vapor-hw secret set staging REDIS_PASSWORD "RedisPass!"
vapor-hw secret set staging RABBITMQ_PASSWORD "RabbitPass!"
vapor-hw secret set staging OBS_KEY "ak"
vapor-hw secret set staging OBS_SECRET "sk"

# 3. Provision semua infra (VPC, RDS, DCS, DMS, APIG, OBS, vhost)
vapor-hw infra apply

# 4. Deploy ke staging dulu
vapor-hw deploy run staging

# 5. Setelah staging OK, deploy production
vapor-hw deploy run production
